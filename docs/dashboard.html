Protocolo de atualização do status (I-DASH)

INVARIANTE I-DASH — status nunca defasa Toda fase que fecha deve bumpar dashboard.html + docs/STATUS.md no mesmo PR. O checkbox do mapa de produção (☐ → ☑) é parte do gate da fase: uma fase não fecha com seu item em ☐. O @orquestrador valida via conector ao revisar o PR — se o status não reflete o HEAD real, o gate falha. Foi a violação deste princípio (catálogo defasado descoberto por releitura) que originou as dívidas A1–A4.

Automação futura (N2 — Sprint 4): scripts/build_dashboard.py lerá STATUS.md + git log + TECHNICAL_DEBT.md e regenerará este HTML automaticamente, substituindo o bump manual. Até lá, a atualização é manual mas obrigatória por invariante — esta é a garantia de que o status acompanha a execução.
