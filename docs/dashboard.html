O que o projeto entrega ao final — visão global

ENTREGÁVEL FINAL (o todo) Uma camada de continuidade formal, instalável abaixo de qualquer agente LLM, que torna falsificáveis [Popper, 1934: testáveis / passíveis de refutação empírica — virtude, não fraudabilidade] as afirmações de progresso — com prova empírica externa e o próprio framework gerenciando seu desenvolvimento (meta-uso). Não é framework de agentes (LangGraph/AutoGen/CrewAI são); é a camada formal abaixo de qualquer um deles.
AS 3 CONTRIBUIÇÕES (da tese RSL, PQMS 9.68)
  1. Unificar Doc-as-Code + BPMN 2.0 + Context Engineering numa arquitetura única implementada — não só proposta.
  2. Arquitetura que neutraliza inflação via gate triplo que ignora score auto-reportado (origem: SOC V3 9.44 reportado vs 4.49 medido).
  3. Demonstrar propriedade fractal (Hausdorff dim 1.0) com convergência por Banach.

  • assembler completo (transition + stream) operável via CLI
  • avaliação objetiva de qualidade (gate anti-inflação)

Protocolo de atualização do status (I-DASH)

INVARIANTE I-DASH — status nunca defasa Toda fase que fecha deve bumpar dashboard.html + docs/STATUS.md no mesmo PR. O checkbox do mapa de produção (☐ → ☑) é parte do gate da fase: uma fase não fecha com seu item em ☐. O @orquestrador valida via conector ao revisar o PR — se o status não reflete o HEAD real, o gate falha. Foi a violação deste princípio (catálogo defasado descoberto por releitura) que originou as dívidas A1–A4.

Automação futura (N2 — Sprint 4): scripts/build_dashboard.py lerá STATUS.md + git log + TECHNICAL_DEBT.md e regenerará este HTML automaticamente, substituindo o bump manual. Até lá, a atualização é manual mas obrigatória por invariante — esta é a garantia de que o status acompanha a execução.
